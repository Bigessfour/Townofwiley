# Town of Wiley AI Workspace Instructions

Use these instructions when generating, editing, or reviewing code in this workspace.

## Project Context

- This repository is an Angular 21 application with standalone components, signals-first patterns, native control flow, strict accessibility expectations, Playwright e2e coverage, and AWS-backed infrastructure.
- Treat `.vscode/angular-best-practices.md` as the source of truth for Angular, TypeScript, accessibility, component, template, service, and responsive-design guidance.
- Treat `.vscode/llms.txt` as the quick Angular context bundle for this workspace.
- Treat `.github/instructions/ai-application-development.instructions.md` as the source of truth for AI feature integration, provider security boundaries, tool calling, and graceful degradation.

## Angular Requirements

- Prefer standalone Angular APIs, signals, `input()`, `output()`, `computed()`, and `inject()`.
- Use `ChangeDetectionStrategy.OnPush` for components.
- Prefer native control flow blocks (`@if`, `@for`, `@switch`) over structural directives.
- Prefer reactive forms over template-driven forms.
- Avoid `ngClass` and `ngStyle`; prefer class and style bindings.
- Keep templates semantic and accessible, and meet WCAG AA requirements.

## AI Tooling Expectations

- For Angular-specific tasks, use the Angular CLI MCP server before falling back to generic framework knowledge when the MCP tools are available.
- Use repo-local guidance before inventing patterns: `.vscode/angular-best-practices.md`, `.vscode/llms.txt`, and `.github/copilot-instructions.md`.
- For AI-powered feature work, follow `.github/instructions/ai-application-development.instructions.md` and route larger AI integration tasks through the `.github/agents/ai-feature.agent.md` agent when appropriate.
- For Playwright work, follow `.github/instructions/playwright-testing.instructions.md`.

## Evaluation And Validation

- Prefer the existing workspace scripts for build, lint, unit tests, Vitest, and Playwright validation.
- Web Codegen Scorer is available in this workspace for Angular-focused AI evaluation through `hello-world/config.mjs`.
- Keep changes minimal and aligned with existing project structure and styling.
